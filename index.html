Matlab client library for PostgreSQL

PgMex enables Matlab-based application to communicated with PostgreSQL database in a Matlab native way passing data in form of matrices, multi-dimensional arrays and structures. The library is written in pure C which results into a very high performance for both small and data-heavy database requests. Both Windows and Linux OSs are supported.

Of course PgMex is not the only way you can communicate with PostgreSQL from Matlab, you could use JDBC or even build a very thin mex wrapper function around C client libraries for PostgreSQL. JDBC solution suffers from higher than libpq-based latency, low performance for large data sets and tuples with multi-dimensional values (PostgreSQL arrays). The second solution would require basically writing PgMex unless you really need a narrow set of functionality comparing to what PgMex can do.
